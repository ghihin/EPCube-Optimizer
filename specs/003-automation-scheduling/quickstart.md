# Quickstart: Automation & Scheduling

本機能の動作確認およびテスト方法について説明します。

## 事前準備

1. **カレンダーアプリの準備**:
   - 開発対象のAndroid端末にGoogleアカウントを登録し、標準のカレンダーアプリで予定が同期される状態にしてください。
   - テスト用に翌日のカレンダーに「【出社】テスト」という予定を作成してください。
2. **権限の付与**:
   - アプリをインストール後、設定画面からカレンダーアクセス権限（`READ_CALENDAR`）を許可してください。
   - 他のアプリの上に重ねて表示する権限、およびアクセシビリティ（ユーザー補助）権限を付与してください。
   - **MIUI等のカスタムOSの場合**: 「ロック画面に表示」「バックグラウンドで実行中にポップアップウィンドウを表示する」「ポップアップウィンドウを表示」の権限を許可し、バッテリーセーバーを「制限なし」に設定してください。
3. **ロック画面の解除**:
   - 端末のセキュリティ設定で、画面ロックを「なし」に設定してください。
   - **MIUI等のカスタムOSの場合**: 開発者向けオプションから「画面ロックをスキップ（Skip screen lock）」をオンにし、電源ボタン押下時にスワイプ画面を挟まず直接ホーム画面が表示される状態にしてください。

## 機能のテスト方法

### 1. スケジュール・予測SOCビューアーの確認
1. EPCube-Optimizer アプリを起動します。
2. メイン画面の「1週間先のスケジュール・予測SOCを確認」ボタンをタップします。
3. 本日から最大7日先までの日付リストが表示されることを確認します。
4. 日付と曜日が「MM/dd (E)」形式で表示され、天気予報（天気、雲量、降水確率）が日本語で表示されていることを確認します。
5. 明日の日付の行が「外出モード」と判定され、季節・天気に応じたSOC予測値が計算表示されていることを確認します。

### 2. 深夜の全自動実行（手動トリガーテスト）
1. 開発中は、23:30の時刻を待たずにテストできるよう、アプリ内に「手動フルオートマクロ実行テスト」ボタン（デバッグ用）を用意しています。
2. または、`Config.kt` の `AUTO_EXECUTION_HOUR` と `AUTO_EXECUTION_MINUTE` を現在時刻の数分後に設定してビルド・インストールします。
3. アプリを一度起動してアラームをスケジュールします。
4. 端末をスリープ状態（電源ボタンを押して画面を暗くする）にします。
5. 設定した時刻になると、端末の画面が自動的に点灯し、UIが操作され、マクロが完了してSOCが設定されるまでを見届けます。
6. ログ（LogcatまたはFirebase）に実行結果（Success）が記録されることを確認します。
