# 機能仕様: EP CUBE マクロアクセシビリティ

**機能ブランチ**: `002-epcube-macro-accessibility`  
**作成日**: 2026-03-31  
**ステータス**: Draft  
**入力**: ユーザーの説明: "EP CUBEの公式アプリには外部連携用のAPIが存在しないため、本アプリ（EPCubeOptimizer）で算出した「目標SOC（充電上限）」を、Androidのユーザー補助機能（AccessibilityService API）を用いてEP CUBEアプリのUIを自動操作することで反映させるマクロ機能を実装する。"

## ユーザーストーリーとテスト *(必須)*

### User Story 1 - EP CUBEアプリの自動操作による目標SOC設定 (Priority: P1)

ユーザーがEPCubeOptimizerアプリで算出した目標SOCを、EP CUBE公式アプリに自動的に反映させる。

**この優先度の理由**: 本アプリのコア機能である「算出した目標SOCを実際の蓄電池に反映させる」ための唯一の手段であり、最も重要な機能であるため。

**独立したテスト**: EPCubeOptimizerからインテントを発行し、EP CUBEアプリが起動して一連のUI操作が自動で行われ、最終的にEPCubeOptimizerに戻ってくることを確認することで独立してテスト可能。

**受け入れシナリオ**:

1. **Given** EPCubeOptimizerアプリで目標SOCが算出されており、EP CUBEアプリがインストールされている状態、**When** マクロ実行を開始する、**Then** EP CUBEアプリが起動し、ホーム画面の「スマートモード」の右側にある設定ボタンが自動でクリックされる。
2. **Given** EP CUBEアプリの運転モード切替画面が表示されている状態、**When** 「充電上限」の現在の設定値を読み取る、**Then** 目標SOCと異なる場合は「＋」「ー」ボタンまたはスライダーを操作して目標値に変更される。
3. **Given** 充電上限が目標値に変更された状態、**When** 画面最下部の「設定」ボタンをクリックする、**Then** 20秒間待機し、設定が反映されるのを待つ。
4. **Given** 20秒経過した状態、**When** 再度「充電上限」の現在の設定値を読み取る、**Then** 目標SOCと一致していれば設定成功と判断し、左上の「戻る」ボタンを押してホーム画面に戻った後、自動的にEPCubeOptimizerアプリに画面遷移して戻る。

### エッジケース

- EP CUBEアプリがインストールされていない場合はどうなるか？
- EP CUBEアプリがログインされていない状態（ログイン画面が表示される）の場合はどうなるか？
- 画面の描画遅延や通信エラーにより、期待するUI要素（「スマートモード」や「充電上限」など）が指定時間内に見つからない場合はどうなるか？
- 「ローディング」後に「成功しました。」以外のエラーメッセージが表示された場合はどうなるか？（※現在は20秒後の再確認で判定するため、エラーメッセージ自体は検知しない）
- ユーザーが自動操作中に手動で画面をタップしたり、別のアプリに切り替えたりした場合はどうなるか？

## 要件 *(必須)*

### 機能要件

- **FR-001**: System MUST AndroidのAccessibilityService APIを用いて、対象パッケージ（`com.eternalplanetenergy.epcube.jp`）のUI要素を監視・操作できること。
- **FR-002**: System MUST 座標指定（X,Y）ではなく、`AccessibilityNodeInfo`を用いたテキストやViewIDベースでUI要素を特定すること。
- **FR-003**: System MUST 画面の描画遅延や通信によるローディング時間を考慮し、各UI要素の特定に適切な待機処理（リトライやタイムアウト）を設けること。
- **FR-004**: System MUST ホーム画面から「スマートモード」のテキストを検索し、関連する設定ボタンをクリックすること。
- **FR-005**: System MUST 運転モード切替画面で「充電上限」の現在値を読み取り、目標値と異なる場合のみUI操作（「＋」「ー」ボタン等）で値を変更すること。
- **FR-006**: System MUST 設定変更後、「設定」ボタンをクリックし、20秒間待機した後に再度「充電上限」の値を読み取って目標値と一致しているか確認すること。
- **FR-007**: System MUST 設定完了を検知した後、左上の「戻る」ボタンを押してホーム画面に戻り、自動的にEPCubeOptimizerアプリに画面遷移して復帰すること。

### 主要なエンティティ *(データが関与する場合)*

- **TargetSOC**: EPCubeOptimizerで算出された充電上限の目標値（0〜100の整数値）。
- **MacroState**: 自動操作の現在の進行状態（例: ホーム画面待機中、設定画面待機中、保存待機中、完了）。

## 成功基準 *(必須)*

### 測定可能な結果

- **SC-001**: 目標SOCの反映マクロが、正常な通信環境下において95%以上の成功率で最後まで完遂すること。
- **SC-002**: マクロ実行開始からEPCubeOptimizerアプリへの復帰まで、ユーザーの追加操作なしで自動的に完了すること。
- **SC-003**: 端末の画面解像度やアスペクト比が異なる複数のAndroidデバイスで、UI要素の特定と操作が正常に機能すること。

## 前提条件

- ユーザーは事前にAndroidの設定画面から、本アプリのユーザー補助機能（AccessibilityService）を有効にしている。
- EP CUBEアプリは既にログイン済みであり、起動直後にホーム画面（または前回の状態）が表示される。
- EP CUBEアプリのUI構造（テキストやボタンの配置）は、提供されたスクリーンショットのバージョンから大きく変更されていない。
- 目標SOCの変更操作において、「＋」「ー」ボタンの1回のタップで変化する値の増減幅は既知である（例: 1%ずつ、または5%ずつ）。
- 本アプリは特定の端末（Redmi 9T）で動作することを想定しており、EP CUBEアプリは同端末にインストール済みである。
