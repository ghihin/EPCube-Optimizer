# Specification - Dynamic Mode Switching (Ver 3.0)

## 目的
現在の「スマートモードでのSOC設定」に加え、アプリ画面から「現在の蓄電池残量（SOC）」をスクレイピング（読み取り）し、その値と翌日の天気予報を組み合わせて、運転モードを「グリーンモード」と「スマートモード」で動的に切り替える。
これにより、日照量が確保できる日において、深夜・早朝の不要な系統買電を避けつつ自家消費の最大化を図る。

## モード切替の判定ロジック
深夜の自動実行時に以下の振る舞いを行う：

* **条件**: 画面から読み取った現在のSOCが「60%以上」 かつ 翌日の天気予報が「晴れ」（例: 雲量<=30% かつ 降水確率<20%など既存条件の流用）
* **アクション**: **【グリーンモード】** に切り替える。
* **理由**: 翌日の太陽光発電で十分に充電可能であり、現在の残量によって夜間の待機電力も十分賄えるため買電をストップする。

* **上記以外の場合** (雨天や、現在のSOCが60%未満の場合、あるいは読取エラー等のフェイルセーフ時):
* **アクション**: **【スマートモード】** に切り替え、従来通りユースケースで算出した目標SOCで指定する。

## マクロ (AccessibilityService) UI操作仕様

### フェーズ1: 読み取り (Scraping)と状態確認
1. EP CUBEアプリ起動後、ホーム画面から正規表現 `\((\d+)%\)` を用いて「カッコ付きのSOC数値」を検索し誤検知（自給率など）を防ぎつつ数値を取得する。
2. 同時に、ホーム画面から「現在の運転モード」のテキストも読み取る。

### フェーズ2: 設定 (Action) と 冪等性(Idempotency)
**パターンA: グリーンモードへの切替**
- 現在すでに「グリーンモード」の場合は、無駄なUI操作を一切行わずマクロ成功として終了する（冪等性の確保）。
- 上記以外の場合：
  1. ホーム画面下部の運転モードボタンをタップ。
  2. 運転モード切替画面で「グリーンモード」ラジオボタンをタップ。
  3. （場合により）「確認」をタップ。
  4. 画面下部の「設定」ボタンをタップ。

**パターンB: スマートモードへの切替 (フォールバック含む)**
- 現在すでに「スマートモード」の場合は、再度のモード選択設定は不要なため、モードボタンタップ後にそのまま「目標SOCスライダーの調整」のみを行う。
- 上記以外（グリーンモード等からの切り替え時）：
  1. ホーム画面下部の運転モードボタンをタップ。
  2. 運転モード切替画面で「スマートモード」ラジオボタンをタップして「設定」で一度確定。
  3. 新しい状態をUIに反映させるため一度戻り、再度スライダー画面を開いて「充電上限」スライダーを目標SOCに合わせて操作し、再度「設定」をタップ。

## 予測スケジュールUI仕様
- **UI表示**: アプリ内の1週間先予測スケジュール一覧にて、各日の予測天気が「晴れ」の条件を満たす場合、対象日のカードに「🌿 グリーンモード対象 (※SOC≥60%時)」という表示を付加する。満たさない日は「🔋 スマートモード予定」とする。
- これにより、今後のモード切り替えの状況をユーザーが一目で把握できるようにする。
